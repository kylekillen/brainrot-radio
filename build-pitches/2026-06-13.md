## PITCH 1 (Primary): Cole Medin's AI Layer — The `.claude/` Directory as a Second Codebase

**Technique:** Every serious AI-augmented project has two codebases: the application code and the *AI layer* — a structured `.claude/` directory containing (a) lean global rules in CLAUDE.md, kept short to fight "context rot," (b) on-demand context docs loaded selectively per task rather than globally, and (c) 15 packaged slash-commands for every recurring workflow. The discipline: CLAUDE.md holds only universal constraints that apply to every session. Task-specific context (beat guidelines, scoring logic, format specs) lives in separate files loaded on demand. Overstuffing CLAUDE.md makes the AI carry irrelevant context into every session, degrading output. Verbose global rules = context rot. The 15 commands (`/plan`, `/implement`, `/validate`, `/review`, `/security-review`, etc.) mean no recurring workflow gets re-explained twice.

**Who's doing it:**
- **Cole Medin** — presented "Advanced Claude Code Techniques: Agentic Engineering With Context Driven Development" at NodeJS community conference, Amsterdam & Online, June 11-15, 2026.
  - Full workshop materials (live GitHub repo with all 15 commands): https://github.com/coleam00/ai-transformation-workshop
  - Conference listing: https://gitnation.com/contents/advanced-claude-code-techniques-agentic-engineering-with-context-driven-development-3256
- **MindStudio** independently documented the PIV loop as a corroborating framework: https://www.mindstudio.ai/blog/agentic-coding-workflow-piv-loop-explained

**Evidence it's real:**
- GitHub repo has working code: `.claude/commands/` with all 15 commands, a demo app, and a structured `.agents/` directory for generated PRDs and stories. Drop-in ready, not a mockup.
- Conference presentation was a hands-on two-hour workshop; attendees implemented the AI layer during the session.
- MindStudio published independent coverage of the PIV loop in the same timeframe, confirming adoption outside Medin's circle.
- The "context rot" diagnosis matches practitioner reports across multiple independent sources (DEV Community, AI Corner guide, tutorials.technology). Not a new coinage — Medin formalized an observed pattern.

**Why it matters for us:**
Kyle's `brainrot-radio/CLAUDE.md` is already very long (~400+ lines). That is textbook context rot — every session loads beat structure, scoring algorithm, voice definitions, dedup rules, publishing config, and the future roadmap, even when the task is just "run QC." Every agent starts carrying the full weight of the CLAUDE.md regardless of what it actually needs.

More concretely: the podcast pipeline has five recurring operations — ingest, beat reporting, assembly, QC, publish — and none are packaged as commands. Every operator re-derives the right prompt from reading CLAUDE.md. If those five steps were `.claude/commands/generate-brief.md`, `/run-beat-reporters.md`, `/assemble-episode.md`, `/qc-episode.md`, and `/publish-episode.md`, any future agent picks up the pipeline cold and runs correctly with no briefing overhead.

The "system gaps" concept is the most directly applicable: when an episode fails QC or a beat agent misses a source, treat it as a bug in the AI layer — update the relevant command — not a one-off fix in the output. Kyle already has this instinct (CLAUDE.md is updated regularly) but without the split structure, fixes pile into global rules and worsen context rot over time.

**Build sketch** (an agent could run this cold from this file):
1. Audit `brainrot-radio/CLAUDE.md` — separate task-specific content (beat editorial guidelines, scoring algorithm, per-beat editorial notes, dedup procedures, pitch format) from true globals (voice names, episode format, mandatory QC rule, file conventions).
2. Create `.claude/commands/` with five episode pipeline commands: each file is the specific prompt + context loading instructions for that step only.
3. Move per-beat editorial guidelines to `.claude/context/beats/` (one file per beat). Commands load only the relevant beat at start.
4. Trim CLAUDE.md to ~100 lines of true globals: voice assignments, format rules, mandatory QC rule, file conventions, known errors.
5. Test: run a cold-start agent with the new structure and compare QC pass rate vs. current.

## PITCH 2: Adversarial Verification QC — Three Refuter Agents Instead of One Checker

**Technique:** The mandatory QC step in the podcast pipeline uses a single agent to find problems. Claude Code's adversarial verification pattern (formalized in dynamic workflows, announced June 2, 2026) replaces one checker with three independent agents each explicitly told to *attack and refute* the script from a different angle. A synthesizer reads all three attack reports and flags only issues two or more agents agree on as must-fix; single-agent findings are advisory. This eliminates both false positives (one overzealous checker) and false negatives (one agent misses what two would catch). The incentive structure is the key: an agent asked to "find problems" tries to be helpful; an agent asked to "try to prove this script is wrong" tries to refute. Different instructions, different effort directed at different failure modes.

**Who's doing it:**
- **Anthropic** — documented in their blog "A harness for every task: dynamic workflows in Claude Code" and the launch post "Introducing dynamic workflows in Claude Code" (June 2, 2026):
  - https://claude.com/blog/a-harness-for-every-task-dynamic-workflows-in-claude-code
  - https://claude.com/blog/introducing-dynamic-workflows-in-claude-code
- **InfoQ** covered real-world adoption: https://www.infoq.com/news/2026/06/dynamic-workflows-claude-code/
- Real-scale demonstration: the Bun project used dual reviewers per file to rewrite 750,000 lines of Rust in 11 days ("eleven days from first commit to merge").

**Evidence it's real:**
- Anthropic named and described adversarial verification explicitly as a pattern, with "self-preferential bias" named as the failure mode it solves. Not vague — this is a documented, live feature.
- The Bun rewrite is a verifiable case study, not a benchmark.
- InfoQ and independent practitioners confirm usage, not just Anthropic promotion.
- The "self-preferential bias" mechanism is independently attested in LLM research: models evaluate their own outputs more favorably than identical outputs from another source.

**Why it matters for us:**
CLAUDE.md already mandates: *"ALWAYS run a QC agent after writing the script, BEFORE rendering. Do NOT render until QC passes clean."* But a single QC agent repeatedly fails to catch: stories covered in a previous episode (dedup failure), editorializing without source grounding, and structural problems only visible when reading the full episode flow. These are exactly the failures that surface post-publish.

The adversarial structure maps directly onto the three hardest QC failure categories for this pipeline:
- **Freshness & Dedup Skeptic** — attacks the script looking *only* for stories covered in previous episodes or marked stale in `.covered-YYYY-MM-DD.json`. Motivated to find every dedup failure; has no stake in the script being good.
- **Coherence Skeptic** — attacks the script looking *only* for structural problems: repeated sign-offs, orphaned content, segments out of order, connections forced between unrelated stories.
- **Sourcing Skeptic** — attacks the script looking *only* for segments missing direct quotes, editorial claims without grounding, "here's what I imagine" language.

A synthesizer reads all three reports and produces: (a) MUST-FIX list (≥2 agents flagged it) and (b) ADVISORY list (1 agent only). The synthesizer replaces the current single-agent QC prompt entirely.

**Build sketch** (an agent could run this cold from this file):
1. Write `.claude/commands/qc-episode.md` — a command that launches 3 adversarial agents in parallel (Agent tool, `subagent_type: "general-purpose"`), each with a focused attack-mode prompt for one of the three categories above. Each agent reads the full script and `.tmp/covered-YYYY-MM-DD.json` for its beat.
2. Add a synthesizer step: a fourth agent reads all three attack reports and produces the MUST-FIX / ADVISORY split with a PASS or FAIL verdict.
3. Update `generate-episode.sh` Step 5 to call this command instead of the current ad-hoc QC prompt.
4. Update the QC section of CLAUDE.md: replace the current description with "adversarial 3-agent QC — see `.claude/commands/qc-episode.md`."
